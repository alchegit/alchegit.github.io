## 목표

기존 가이드는 1화 전체의 20개 장면을 한 번에 보여줘 초보자가 결과를 읽고 판단하기 어려웠다. 이번 테스트에서는 같은 세계관의 연속된 세로 이미지 6장을 골라 짧은 도입부로 다시 편집하고, 매 단계에서 `관찰 → 판단 → 수정`을 거친 뒤 다음 작업을 진행했다.

가이드는 저장된 샘플 이미지를 사용하므로 실제 이미지 생성 서버를 호출하거나 도토리를 차감하지 않는다. 아래 시간은 브라우저의 가이드 연출과 편집 기능이 끝날 때까지의 실제 측정값이다. 실제 이미지 생성 시간은 공급자와 서버 상태에 따라 별도로 측정해야 한다.

## 최종 6컷 설계

| 컷 | 장면 | 카메라·인물 방향 | 이야기 역할 |
|---|---|---|---|
| 1 | 막차 뒤의 정적 | 눈높이·뒷모습 | 평범한 야간 근무와 고립을 한 컷에 제시 |
| 2 | 천장의 균열 | 로우앵글·오른쪽 반측면 | 일상이 깨지는 사건의 규모 강조 |
| 3 | 첫 임무 | 탑뷰·앞모습 | 인물이 시스템에 갇힌 공간 관계 표현 |
| 4 | 벽 너머 무전 | 측면·왼쪽 반측면 | 구조 대상의 존재와 이동 방향 제시 |
| 5 | 한 사람의 출구 | 바닥 로우앵글에서 탑뷰로 수정 | 힘 있는 결심보다 선택을 강요받는 상황을 먼저 전달 |
| 6 | 첫 선택 | 어깨 너머·뒷모습 | 위협과 주인공을 함께 보여주며 다음 장면 훅 형성 |

## 단계별 실행 로그

### 1. 원고 입력과 진단

- 실제 소요: 0.826초
- 관찰: 폐역, F급 헌터와 역의 규칙은 선명하지만 짧은 가이드에서는 구조 대상과 주인공의 선택이 늦게 드러났다.
- 판단: 전체 1화를 압축하지 않고, 갈등과 선택이 한 번에 보이는 6컷 도입부로 범위를 줄였다.
- 수정: `일상 → 균열 → 임무 → 구조 신호 → 실패 조건 → 규칙 거부` 순서로 재배치하고 마지막을 “둘 다 나간다”는 선언으로 끝냈다.

### 2. 6컷 시안 생성과 검토

- 실제 소요: 5.761초
- 관찰: 눈높이 구도만 이어지면 균열의 규모와 주인공의 고립감이 약해졌다.
- 판단: 컷마다 정보 목적을 하나씩 배정하고 같은 각도를 연속 사용하지 않기로 했다.
- 수정: 로우앵글, 탑뷰, 측면, 바닥 로우앵글과 어깨 너머 구도를 순서에 맞게 배치했다.

### 3. 선택 시안 완성화와 안전 영역 검수

- 실제 소요: 5.160초
- 관찰: 인물 외형과 폐역 조명은 이어졌지만 글자가 얼굴과 푸른 상태창을 가릴 가능성이 있었다.
- 판단: 그림에 글자를 합성하지 않고 말풍선, 나레이션과 효과음을 독립 객체로 유지했다.
- 수정: 완성 직후 세 글자 요소를 각각 수정하고 이동할 수 있도록 편집 상태로 자동 전환했다.

### 4. 말풍선·나레이션·효과음 편집

- 실제 소요: 1.628초
- 관찰: 5컷의 원래 말풍선과 효과음이 얼굴과 상태창이 있는 중앙 시선을 방해했다.
- 판단: 문장을 짧게 고치고 세 요소를 서로 다른 안전 영역으로 옮겼다.
- 수정: 말풍선은 상단 30%, 나레이션은 하단 5%, 효과음은 오른쪽 68% 지점으로 이동했다.

### 5. 도토리 안내 후 선택 컷 재연출

- 실제 소요: 2.658초
- 관찰: 5컷의 낮은 시점은 결심을 강조했지만 시스템에 갇힌 인물과 공간의 관계는 충분히 보이지 않았다.
- 판단: 5컷만 탑뷰·앞모습으로 다시 만들고 다른 컷은 유지했다.
- 수정: 새 버전을 현재 컷에 적용하되 기존 5컷은 장면 보관함에 남겼다. 가이드에서는 무료이며 실제 작업에서만 도토리 1개를 확인한다.

### 6. 보관 장면 재사용

- 실제 소요: 0.039초
- 관찰: 1컷 이미지로 현재 5컷을 덮으면 사건의 인과가 끊기고 새 구도도 사라졌다.
- 판단: 교체가 아니라 마지막 결심 직전의 회상 컷으로 쓰는 편이 이야기 목적에 맞았다.
- 수정: 원본과 현재 컷을 보존한 채 `reusedFrom: p1` 출처가 있는 회상 컷을 삽입했다. 최종 결과는 7컷이다.

## 결론

시나리오 테스트는 통과했다. 가이드가 단순 기능 시연이 아니라 결과를 보고 이유를 설명한 다음 필요한 부분만 고치는 작은 제작 사례로 바뀌었다. 장면 재사용도 기존 컷을 무조건 교체하지 않고 이야기 목적에 따라 회상 컷으로 삽입한다.

실제 생성 플랫폼에서 추가로 검증해야 할 항목은 이미지 공급자의 생성 시간, 생성 실패 재시도, 실제 도토리 원장 차감과 서버에 저장된 이미지 버전의 재사용이다.
